Accessibility Auditor Release Notes: Version 2.0.4

A list of featured updates to the HTML_CodeSniffer "Accessibility Auditor" bookmarklet in release version 2.0.4, released on 2nd February 2015. This list is focused for users of the bookmarklet, and focuses on changes to tests and the auditor interface. A full list of updates can also be found in the project's technical bookmarklet changelog.

WCAG 2.0 Standards

Items referring to Success Criteria and Guidelines should be read in conjunction with the W3C's documentation for the Web Content Accessibility Guidelines (WCAG) 2.0.

Standards updates

  • Success Criteria 1.3.1: If a "for" attribute on a label element is filled but does not point to the ID of a form control, this is now treated as a Warning rather than an Error. It is not seen as crucial that the association is not made, however if the "for" attribute has been filled, the intention is assumed that it is meant to link to a form control - and thus if it doesn't, it should be manually checked for potential mistyping or other review.
  • Success Criteria 1.4.3, 1.4.6: Elements which are marked as disabled using the "disabled" or "aria-disabled" attributes will now be ignored for colour contrast purposes.
