Accessibility Auditor Release Notes: Version 2.0.2

A list of featured updates to the HTML_CodeSniffer "Accessibility Auditor" bookmarklet in release version 2.0.2, released on 12th November 2014. This list is focused for users of the bookmarklet, and focuses on changes to tests and the auditor interface. A full list of updates can also be found in the project's technical bookmarklet changelog.

WCAG 2.0 Standards

Items referring to Success Criteria and Guidelines should be read in conjunction with the W3C's documentation for the Web Content Accessibility Guidelines (WCAG) 2.0.

Standards updates

  • Success Criterion 1.3.1: The portion of this standard that handled labels on inputs has been updated for changes to the appropriate technique made in March 2014. This includes:
    • permitting label elements to wrap around inputs, instead of simply "explicit" referencing using the for attribute;
    • recognising aria-label and aria-labelledby attributes, as well as title;
    • recognising the HTML5 hidden attribute;
    • codes for errors now reference common failure technique F68, to make clear that it is a failure.
  • Success Criterion 1.3.1: label elements with duplicate for attributes no longer emit a message, as there seems to be nothing forbidding this.
  • Success Criterion 4.1.2: When a single-selection select element does not contain an option with the selected attribute, this is now treated as a warning rather than an error. Different versions of HTML have differing viewpoints: in HTML 4 the value of such a select box is undefined, whereas in HTML5 it is set to the first option.
  • Success Criterion 4.1.2: Anchors (a elements) are permitted to count title attributes as valid accessibility API names.
  • Guideline 1.4: If an element is set to be absolutely positioned, or have an alpha channel set (using the rgba() colour format), contrast errors are reduced to warnings.
  • Guideline 1.4: If a colour contrast ratio test fails, but by less of a margin than the default precision (2 decimal places) HTML_CodeSniffer can show, then extra decimals will be shown. (An example in the issues list had a contrast ratio of 4.497:1 which would show as 4.5:1 and thus appearing to "pass" a 4.5:1 contrast test, but now shows with all three decimals.)
